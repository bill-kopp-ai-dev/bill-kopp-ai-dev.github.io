    🚀 Executive Vision

    I have been working with AI Agent Engineering, always focusing on the architecture, development, and deployment of autonomous systems that are secure, scalable, and ready for the corporate environment.

    With a Ph.D. in Industrial Biotechnology and over 15 years of experience leading the development and optimization of industrial biotechnological processes, I developed a deep scientific rigor focused on complex analyses, pattern identification, and continuous learning. Today, I translate this same technical discipline into the AI Engineering ecosystem, working to solve one of the biggest challenges in today's industry: bringing governance and deterministic reliability to probabilistic models (LLMs).

    I believe that the true value of Artificial Intelligence lies not only in achieving high accuracy in a controlled environment, but in how these models are integrated via MLOps pipelines, monitored, and governed in production. My absolute technical focus is at the intersection between Agentic AI and Information Security (Security-first AI), ensuring robust security rails for real-world applications.

    🛠️ My Preferred Stack for Building AI Agents

    • Python to write my code.
    • Pydantic to make sure it works.
    • PydanticAI to create my agents.
    • MCP protocol with FastMCP to give these agents capabilities.
    • Prompt Engineering and Skills Engineering to orchestrate complex workflows.
    • ChromaDB to persist my agents in production, with RAG Knowledge Graphs and BM25.
    • A2A protocol for multi-agent system orchestration.
    • Logfire for observability and real-time monitoring of my agents.
    • Docker to deploy these agents securely and scalably in production.
    • Docker MCP Toolkit to efficiently coordinate MCP servers encapsulated in containers.
    • Docker Agent and Docker A2A to efficiently coordinate multi-agent systems emulated in containers.
    • Google Cloud Platform (GCP) to host my agents in production.
    • Edge Computing to deploy my agents in local environments.

    Note: Some of these tools and frameworks I master, others I'm learning. But regardless, I have been using all of them to develop, daily.
